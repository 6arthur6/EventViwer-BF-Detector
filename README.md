# EventViwer-BF-Detector

A simple Python script that checks Windows Security logs and tries to find possible brute force login attempts.

It reads failed login events directly from Windows Event Viewer and creates an alert when the same source has too many failed logins in a short period of time.

## What It Checks

The script checks:

* Windows Security event `4625`
* failed login attempts
* failed logins grouped by source IP
* local failed logins without an IP, if enabled

By default, it creates an alert when the same source has:

```text
5 failed logins in 10 minutes
```

## Requirements

You need:

* Python 3.10 or newer
* Windows
* PowerShell available
* permission to read the Windows `Security` event log

In most cases, you need to run the terminal as Administrator.

You can also use a user that is part of the `Event Log Readers` group.

## How to Use

To read directly from Windows Event Viewer:

```bash
python3 detector.py
```

To include local failed logins that do not have a source IP:

```bash
python3 detector.py --include-local
```

To search a bigger time range:

```bash
python3 detector.py --hours 48 --max-events 5000
```

To use a JSON file instead of reading from Event Viewer:

```bash
python3 detector.py --source json --events-file events.json
```

## Output

The alerts are saved in:

```text
alerts.json
```

The script also prints a small summary in the terminal, for example:

```text
analysis done.
event source: windows
events checked: 100
4625 failures found: 8
4625 failures used: 8
4625 local/no-ip failures skipped: 0
alerts created: 1
```

## Helper Script

You can use `check_events.py` to inspect a local `events.json` file:

```bash
python3 check_events.py
```

It shows:

* total number of events
* the first event in the file
* a summary grouped by event ID

## Notes

* `alerts.json` is generated by the script, so it should not be committed.
* `events.json` can contain real usernames, hostnames, and internal IPs.
* Use `events.example.json` if you need public sample data.
* To read the Windows Security log, run the script as Administrator if you get a permission error.
